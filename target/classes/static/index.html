How to Get Credentials

  1. Log in to your required website account
  2. Open Browser Developer Tools (Press F12)
  3. Navigate to the Network tab
  4. Refresh the page or perform any action
  5. Click on any network request from the list
  6. Look for Request Headers section
  7. Copy the Cookie header value
  8. Copy the X-ZCSRF-TOKEN header value if required
  9. Copy the User-Agent header value if required

Executing Tests

  1. Navigate to Execute Tests page
  2. Click Upload CSV and select your prepared file
  3. Select one or more test suites from the list
  4. Click Run Tests to begin execution
  5. Monitor live execution logs and real-time progress
  6. View overall summary and account-level results and test-suite level summary
  7. Download reports in PDF format

Check Types

Pattern Match

Validates whether a given pattern is present for the target variable using regular expressions.

Required: Path, Regex Pattern

Key Presence Validator

Checks if a specific element is present at the given path in the response.

Required: Path only

Field Existence

Verifies if specified fields are present at the given path location.

Required: Path, Fields
Optional: fieldsAny

Value Match

Validates the response value at the specified path using flexible comparison rules.

Required: Path, Expected Value(s)
Optional: Operator, Logic, Expected Type

Understanding Path, Field, Expected & fieldAny

When creating test cases, you will enter values like Path, Fields, Expected, and fieldAny. Here's a simple explanation of what each one means.

1. Path

The path tells the system where exactly to look inside the API JSON response. It works like giving directions: ‚ÄúGo inside this object ‚Üí inside this array ‚Üí pick this item ‚Üí read this field.‚Äù

Example: 

Path: data.items[0].id
Meaning: Inside "data" ‚Üí open "items" ‚Üí pick first item ‚Üí read "id"

2. Fields

Fields are a list of keys that MUST exist at the given path. If even one is missing, the check fails.

Example: 

Fields: name, icon, displayName
Meaning: All 3 fields must be present at the specified path.

3. Expected

Expected is the value you want to compare with the actual API response. This can be a number, text, boolean, or even full JSON.

Example: 

Path: status.code
Expected: 200
Meaning: The value at "status.code" must be exactly 200.

4. fieldAny

fieldAny is a list of optional fields. At least ONE of them must exist for the test to pass.

Example: 

fieldAny: email, phoneNumber
Meaning: At least one of these fields must exist.

5. Operator

Defines how the actual value should be compared with the expected value.

Example Operators:
== (Equal), != (Not Equal), >, <, >=, <=,
CONTAINS, NOTCONTAINS, STARTSWITH, ENDSWITH

6. Logic

Used when multiple expected values are given. Determines how checks are evaluated.

AND ‚Üí All expected values must match
OR  ‚Üí Any one match is enough

7. Expected Type

Ensures the actual value returned from API matches the expected data type.

Supported Types:
string, number, boolean, array, object, null

JSON Path Cheat Sheet

Use these simple rules to write correct JSON paths when creating checks. These 7 syntaxes cover 95% of all real test cases.


1. $
   ‚Üí Refers to the entire JSON.

2. a.b
   ‚Üí Go inside nested fields (like folders).
     Example: user.profile

3. $.field
   ‚Üí Field directly under root.
     Example: $.status

4. array[0]
   ‚Üí First item in a list/array.
     Example: pages[0]

5. a.b[0].c
   ‚Üí Nested object ‚Üí array index ‚Üí field.
     Example: data.items[0].id

6. [key=value]
   ‚Üí Find array item where a field matches.
     Example: pages[displayName=Manager]

7. [key="value"]
   ‚Üí Same as above but for text/string values.
     Example: users[name="John Doe"]

üí° Master these basic patterns and you can create almost any test case path with confidence.

Multi-level Test Organization

A test case can depend on another test. Child runs only when Parent passes.

  • Multiple parents supported
  • Used when later tests require values from earlier API calls

Requires Field

Allows extracting values from the parent response for child execution.

Example:
name: accessToken
path: $.auth.token

If parent test fails or value not found ‚Üí the child test is marked as SKIPPED.
